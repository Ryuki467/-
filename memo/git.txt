git バージョン管理ツール
修正などの履歴が残る
バグが発生した場合、原因を特定しやすい

バージョン管理
集中型　１つのリポジトリで集中管理 conflict
例）excel

分散型
自分の手元でリポジトリのコピーを作成して、
その中で作業場所を作る　二段階
自分の手元のリポジトリは大本に反映する前に色々と整理する
まずリポジトリ

用語
branch(枝)
リポジトリには自由に名前を付けることができる
例）master,develop
ブランチを変えるとフォルダ構成を変えずにファイルの中身がすり替わる
clone
リポジトリからファイルを取り込む（初回）
pull
リポジトリからファイルを取り込む（更新）
push
リポジトリにファイルを反映する
checkout
ブランチを変更する
merge
ブランチに別のブランチを取り込む
commit
変更したファイルを記録する
rebase
記録を変更する
